# assets — 실제 이미지 넣는 곳

랜딩(`index.html`)은 아래 파일이 **있으면 자동으로 실제 사진으로 교체**하고,
**없으면 기본 SVG/그라데이션 플레이스홀더를 그대로** 보여줍니다.
(교체 로직: `main.js`의 `applyHeroImage()` / `applyReviewPhotos()`)

## 넣어야 할 파일 3개

| 파일명 (이 폴더에 저장) | 쓰이는 위치 | 원본 (claude.ai/design 프로젝트의 경로) | 권장 비율 |
|---|---|---|---|
| `hero.png` | 히어로 배경 | `uploads/pasted-1782873846462-0.png` | 가로형, 최소 1600px 폭 |
| `review-1.png` | 후기 · 김O은 | `uploads/pasted-1782874019108-0.png` | 가로형 |
| `review-2.png` | 후기 · 이O진 | `uploads/pasted-1782873997356-0.png` | 가로형 |

> 확장자를 `.jpg` 등으로 바꾸려면 `index.html`의 `data-bg`와 `main.js`의 `photo:` 값만 맞춰 수정하세요.

## ⚠️ 해상도 주의 (히어로)

히어로 배경은 전체 폭(약 1130px, 레티나 2x면 ~2260px)을 `cover`로 채웁니다.
원본이 이보다 작으면 브라우저가 확대해 **깨져(픽셀·블록) 보입니다.**

- 권장: **가로형, 폭 1600px 이상** (레티나 대응이면 2000px+)
- 지금 넣으신 `hero.png`는 736×1596(세로형)이라, 가로 1600으로 고품질 업스케일 후
  **상단 나무 캐노피가 프레임처럼 들어오도록** 밴드(1600×1250)로 크롭·최적화했습니다.
  프레이밍은 `main.js`의 `background-position: center 20%` 로 조정합니다.
  확대 아티팩트는 줄었지만 원본 화소가 736이라 완전히 선명하진 않습니다.
- 더 선명하게: claude.ai/design에서 **원본(고해상도)** 을 받거나, 폭 1600px 이상
  가로 사진을 `hero.png`로 덮어쓰면 됩니다.

## 왜 자동으로 안 받아졌나

design MCP(`DesignSync.get_file`)는 파일당 **256 KiB**까지만 반환합니다.
위 3개 원본이 모두 이 한도를 넘어 잘린 채(PNG `IEND` 누락)로만 받아져, 온전한 파일로 저장할 수 없었습니다.

## 가져오는 방법

1. claude.ai/design 에서 이 프로젝트 열기
   → `맑음-새집증후군-랜딩` 열고 이미지 우클릭 저장, 또는 `uploads/` 원본 다운로드
2. 위 표의 파일명으로 이 `assets/` 폴더에 저장
3. `index.html`을 새로고침 — 사진이 자동 반영됩니다

`_design-draw-reference.png` 는 프로젝트에 있던 유일하게 온전히 받아진 이미지(랜딩 미사용, 참고용)입니다.
